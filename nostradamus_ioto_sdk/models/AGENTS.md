# models — Pydantic Data Models

## OVERVIEW

Pydantic v2 models for API request/response serialization. One file per domain entity. All inherit from custom `BaseModel`.

## STRUCTURE

| File | Entity | Models |
|------|--------|--------|
| `_base.py` | — | `BaseModel` (custom Pydantic base) |
| `organization.py` | Organization | `OrganizationResponse`, `OrganizationUpdateRequest` |
| `project.py` | Project | `ProjectResponse`, `ProjectCreateRequest`, `ProjectUpdateRequest` |
| `collection.py` | Collection | `CollectionResponse`, `CollectionCreateRequest`, `CollectionUpdateRequest` |
| `project_key.py` | API Key | `ProjectKeyResponse`, `ProjectKeyCreateRequest`, `BaseKeyModel` |
| `data.py` | Data ops | `DeleteDataRequest` |
| `enums.py` | Enums | `KeyType` (read/write/master), `StatOperation` (avg/max/min/sum/count/distinct) |
| `errors.py` | Error models | `HTTPValidationError`, `ValidationError` |
| `_generated.py` | Auto-gen | **DO NOT EDIT** — generated by `scripts/generate_models.py` |

## CONVENTIONS

- **Inherit** from `models._base.BaseModel`, NOT directly from `pydantic.BaseModel`
- BaseModel config: `use_enum_values=True`, `validate_assignment=True`, `populate_by_name=True`
- BaseModel provides JSON encoders for `UUID` → `str` and `datetime` → ISO format
- **Naming**: `{Entity}Response` for API responses, `{Entity}CreateRequest` / `{Entity}UpdateRequest` for mutations
- **Exports**: All public models must be added to `__init__.py` `__all__`
- **Package root export**: After adding to `models/__init__.py`, also add to `nostradamus_ioto_sdk/__init__.py`

## ADDING A NEW MODEL

1. Create `{entity}.py` — define request/response models inheriting `BaseModel`
2. Add imports to `models/__init__.py` and `__all__`
3. Add imports to `nostradamus_ioto_sdk/__init__.py` and `__all__`
4. Create corresponding resource in `resources/`

## ANTI-PATTERNS

- **NEVER** edit `_generated.py` — run `scripts/generate_models.py` to regenerate
- **NEVER** use `pydantic.BaseModel` directly — always use `models._base.BaseModel`
- **NEVER** add `json_encoders` per model — they're defined once in `_base.py`
